# Flathub — human-only submission gate

Do not submit AI-generated material from this distribution kit to Flathub.
Flathub's requirements page, checked 2026-07-20, says submission pull requests,
review comments, descriptions, and applications containing AI-generated or
AI-assisted content are not allowed. A human maintainer must decide whether and
how to proceed, author the submission, and confirm the complete project's
eligibility under that policy.

## Audit of the existing upstream manifest

The existing `apps/desktop/flatpak/com.parsonlabs.Parson.yml` is an early local
prototype and is not submission-ready:

- It installs a prebuilt `Parson.AppImage`; current Flathub rules require open-
  source applications to be built entirely from source without network access.
- The source uses a local `path`, while a submission needs pinned public sources
  and SHA-256 checksums.
- Required upstream AppStream MetaInfo, desktop file, and correctly named icon
  are not present.
- Electron, Bun/npm, Rust/Cargo, and other dependencies need offline source
  manifests generated by accepted tooling.
- The runtime must be the latest supported Flathub runtime at submission time.
- `com.parsonlabs.Parson` requires control of and an HTTPS relationship to
  `parsonlabs.com`; otherwise a human should select a compliant code-hosting ID
  before release and keep the app ID, desktop ID, and MetaInfo ID identical.
- Permissions must be minimized and justified. Network and audio are needed;
  file access should use the file chooser portal instead of broad host access.
- The application must prove it is more than a thin web wrapper and remains
  functional within the Flatpak sandbox, including the bundled Rust server,
  music-folder selection, playback, and persistent data.

## Human validation checklist

After a human authors the source build and metadata:

```bash
flatpak install -y flathub org.flatpak.Builder
flatpak remote-add --if-not-exists --user flathub https://dl.flathub.org/repo/flathub.flatpakrepo
flatpak run --command=flathub-build org.flatpak.Builder --install <manifest>
flatpak run <app-id>
flatpak run --command=flatpak-builder-lint org.flatpak.Builder manifest <manifest>
flatpak run --command=flatpak-builder-lint org.flatpak.Builder appstream <app-id>.metainfo.xml
flatpak run --command=flatpak-builder-lint org.flatpak.Builder repo repo
```

The human submission goes to the `new-pr` branch of `flathub/flathub`, not
`master`, and must include only the top-level manifest and permitted dependency
files—not source code or binaries. This directory intentionally contains no
submission manifest or PR copy.
